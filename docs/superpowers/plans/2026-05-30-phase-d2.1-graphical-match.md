# Phase D2.1 — Minimal Playable Graphical Match Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring up an `aplicacion` binary that opens a native IrrlichtMt window and plays a full AI-vs-AI Quassimodo match on a real 3D board (board + cells + pawns + barriers, rendered from real `Reglas` game state via the existing `Grafico` classes), driven by two Python agents given on argv — ending at "Hay un ganador!". No menu, instant piece moves, fixed camera. Mirrors the D1 pattern (lean `consola` runner) for graphics.

**Architecture:** A **lean graphical driver** reuses `Grafico::Skin` + `PartidaGrafica` (the existing graphical-match logic: builds the scene, loads agents via `Scripting::Manejador`, applies moves to scene nodes) behind a simple `device->run()` loop with one fixed overhead camera. We do **not** port the menu-centric `Aplicacion`/`ManejadorJuego`/`ManejadorGUI`/`Menu`/`GUI`/`EventReceiver` (deferred to D2.2), and we do **not** port the deleted-from-the-fork animator subsystem — piece/barrier moves are instant `setPosition` (the mechanism D2.0 proved). The `Grafico` library is compiled from an **explicit file subset** that excludes the menu and animator files.

**Tech Stack:** IrrlichtMt 1.9.0mt15 (in flake), the project's pybind11 Python-embedding (D1′, unchanged), assimp-converted `.obj` assets (D2.0), CMake+Ninja, clang 16, macOS frameworks (Cocoa/OpenGL/IOKit).

**Branch:** `feat/d2.1-graphical-match`, cut from `main` (which has the merged D2.0 baseline `d3d1df2`).

**Companion docs:** spec [`docs/superpowers/specs/2026-05-29-d2-graphical-port-design.md`](../specs/2026-05-29-d2-graphical-port-design.md); D2.0 findings in [`.wolf/cerebrum.md`](../../../.wolf/cerebrum.md); the proven render path in [`spikes/d2_render/`](../../../spikes/d2_render/).

## Critical gotchas (every subagent MUST read)

1. **`nix` is NOT on PATH in non-interactive shells.** Prefix every build/run command:
   `. /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh && <command>`. Build via `nix develop -c bash -c '...'`.
2. **Never pipe a running binary through `| tail`/`| head`** — masks the exit code (the D1 SIGTRAP lesson). Capture to files + `echo "exit=$?"`.
3. **No AI/LLM attribution** anywhere — commits, PR, code comments. Plain `git commit` (repo-local `fabzter` identity already pinned).
4. **Do not amend/rewrite history. Do not push** until the PR task. Use **explicit `git add <files>`** — never `git add -A` (the `.wolf/buglog.json` auto-churns; don't sweep it into source commits).
5. **This is a PORT.** Do NOT change game logic in `Reglas`/`Scripting`/`Partida`. Adapt graphical/engine API calls only. If a NEW IrrlichtMt API gap appears that isn't covered below, **STOP and report BLOCKED** with the exact error — do NOT stub game logic or invent a mock to force a build.
6. **`consola` must keep working.** It's the D1/D1′ regression baseline. Verify after build changes.
7. **VERIFIED IrrlichtMt API facts (from D2.0 — use these, do not deviate):**
   - Mesh load: `device->getFileSystem()->createAndOpenFile(path)` → `smgr->getMesh(rf)` → `rf->drop()`. NO `getMesh(const char*)`. `getFileSystem()` is on `IrrlichtDevice`, NOT `ISceneManager`.
   - `setMaterialFlag` / `E_MATERIAL_FLAG` / `EMaterialFlags.h` REMOVED → mutate `SMaterial` fields (`mat.Lighting`, `mat.NormalizeNormals`, `mat.BackfaceCulling`) via `node->forEachMaterial([](video::SMaterial& m){...})`.
   - Dynamic lights REMOVED (`addLightSceneNode`/`ILightSceneNode`) → only `setAmbientLight`. For visible flat color: `Lighting=true` + per-material `EmissiveColor` + `setAmbientLight(black)` (D2.0 recipe) — OR for textured meshes just leave default material (textures show).
   - Particle systems REMOVED (`IParticleSystemSceneNode`, `addParticleSystemSceneNode`, emitters/affectors).
   - Shadow volumes REMOVED (`IShadowVolumeSceneNode`, `addShadowVolumeSceneNode`).
   - **Animator subsystem ENTIRELY REMOVED:** no `ISceneNodeAnimator`, `addAnimator`, `getAnimators`, `removeAnimators`, `createFlyStraightAnimator`, `createCollisionResponseAnimator`, `ESNAT_*`, `ISceneNodeAnimatorCameraMaya.h`.
   - `irrList.h`/`core::list` REMOVED → `std::list`. `irrMap.h`/`core::map` REMOVED → `std::map`.
   - Terrain/skydome REMOVED (`addTerrainSceneNode`, `addSkyDomeSceneNode`, `ITerrainSceneNode`, `createTerrainTriangleSelector`, `getVideoModeList`/`IVideoModeList`).
   - Driver enums: only `EDT_NULL`, `EDT_OPENGL`, `EDT_OPENGL3` exist. `EDT_SOFTWARE`/`EDT_DIRECT3D8`/`EDT_DIRECT3D9`/`EDT_BURNINGSVIDEO` REMOVED.
   - `#include <irrlicht/XXX.h>` (subdir) → `<XXX.h>` (flat). `<irrlicht.h>` umbrella still works.
   - `createDevice(EDT_OPENGL, dimension2d<u32>(W,H), bits, fullscreen, stencil, vsync, receiver)` works. `getTexture(const char*)` works. Screenshot: `createScreenShot()`.
8. **Runtime staging (same as `consola`/D1′):** the Python embedder hardcodes `./lib` on `sys.path` and imports `Reglas.so`. Before running `aplicacion`, stage `lib/Reglas.so` (symlink to `build/src/AgenteWrapper/Reglas.so`) and run from the **repo root** (so `conf/skin_default/*.obj` and `./lib/Reglas.so` resolve).

```cpp
    std::string agent1 = (argc > 1) ? argv[1] : "bin/agenteCamina.py";
    std::string agent2 = (argc > 2) ? argv[2] : "bin/agenteCamina2.py";

    Opciones::ManejadorOpciones opciones(argc, argv);

    // Device (OpenGL; the engine spike + D2.0 proved this on macOS arm64).
    IrrlichtDevice* device = createDevice(
        video::EDT_OPENGL, core::dimension2d<u32>(1024, 768),
        16, false, false, false, nullptr);
    if (!device) { std::cerr << "createDevice failed\n"; return 1; }
    device->setWindowCaption(L"Quassimodo");
    video::IVideoDriver*  drv  = device->getVideoDriver();
    scene::ISceneManager* smgr = device->getSceneManager();
    gui::IGUIEnvironment* env  = device->getGUIEnvironment();

    // Flat-color/ambient recipe (no dynamic lights in the fork).
    smgr->setAmbientLight(video::SColorf(1.0f, 1.0f, 1.0f));  // full ambient so textures show

    // Skin loads all meshes/textures from opciones.conf (.obj models).
    Grafico::Skin* skin = new Grafico::Skin(smgr, env, device->getFileSystem(), opciones);

    // The graphical match (builds Tablero + pieces + (stubbed) torches).
    PartidaGrafica* partida = new PartidaGrafica(smgr, skin, env, /*velAnim*/ 1);

    try {
        partida->SetJugadores(agent1, agent2);   // loads Python agents (Scripting)
        partida->iniciarPartida();
    } catch (std::exception& e) {
        std::cerr << "Error iniciando la partida: " << e.what() << "\n";
        device->drop(); return 2;
    }

    // One fixed overhead camera framed on the board centre.
    scene::ICameraSceneNode* cam = smgr->addCameraSceneNode();
    core::vector3df centro = partida->getCentro();
    cam->setTarget(centro);
    cam->setPosition(centro + core::vector3df(0, 900, -600));  // tune via vision check

    bool announced = false;
    while (device->run()) {
        // Drive the match: with instant moves, animacionesEnd() is always true.
        if (partida->estaEnCurso() && partida->animacionesEnd()) {
            try { partida->siguienteJugada(); }
            catch (std::exception& e) {
                std::cerr << "Error en jugada: " << e.what() << "\n"; break;
            }
        }
        drv->beginScene(true, true, video::SColor(255, 40, 40, 70));
        smgr->drawAll();
        env->drawAll();
        drv->endScene();

        if (partida->hayGanador() && !announced) {
            std::cout << "Hay un ganador! Jugador "
                      << partida->getJugadorGanador() << std::endl;
            announced = true;   // keep the window open showing the final board
        }
    }

    delete partida;
    delete skin;
    device->drop();
    return 0;
}
```
