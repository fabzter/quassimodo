# Phase D2.0 — Render-from-Real-State Spike Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove, in an isolated spike under `spikes/d2_render/`, that IrrlichtMt 1.9.0mt15 on macOS arm64 can (a) load Quassimodo's models after a `.3ds`→`.obj` conversion, (b) render a scene derived from a **real** `Reglas::Tablero` (board + 81 cells + 2 pieces + 1 barrier), and (c) reflect one **real** move (`Tablero::moverJugador`) on screen — settling the asset pipeline and the per-move scene-update path before the full D2.1 port.

**Architecture:** A standalone `d2_render` binary links the existing shared `Reglas` lib (single RTTI source) + `IrrlichtMt` + macOS frameworks. A minimal `scene_builder` maps a real `Reglas::Tablero` to an IrrlichtMt scene tree. **No** `Grafico::` types, **no** `Skin`/config machinery, **no** Python embedding, **no** menu — those are D2.1. This mirrors how the D1.5 `pybindspike` stand-ins proved binding mechanisms without the real `Reglas` graph.

**Tech Stack:** IrrlichtMt 1.9.0mt15 (in flake), assimp 6.0.4 (added to flake for one-time mesh conversion), CMake 3.30 + Ninja, clang 16, macOS frameworks (Cocoa, OpenGL, IOKit). Reglas built by the existing `src/Reglas` CMake target.

**Branch:** `feat/d2.0-render-spike`, cut from `main`.

**Companion spec:** [`docs/superpowers/specs/2026-05-29-d2-graphical-port-design.md`](../specs/2026-05-29-d2-graphical-port-design.md) (§3 asset pipeline, §4 the spike). Read it before starting.

**Engine spike reference:** `spikes/engine/main.cpp` — the proven IrrlichtMt window pattern on macOS arm64 (`createDevice(EDT_OPENGL, ...)`, 120-frame loop).

## Critical gotchas (every subagent MUST read this section)

1. **`nix` is NOT on PATH in non-interactive shells.** Prefix every nix command:
   `. /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh && <command>`. Build via `nix develop -c bash -c '...'`.
2. **Nix flakes only see git-tracked/staged files.** After editing `flake.nix`, `git add flake.nix` BEFORE `nix develop`/`nix flake lock`, or the change is invisible.
3. **Never pipe a running binary through `| tail`/`| head`** — it masks the exit code (the D1 SIGTRAP-masking lesson). Capture stdout/stderr to files and `echo "exit=$?"` separately.
4. **Never amend or rewrite history.** Commit per task with a focused message.
5. **No AI/LLM attribution anywhere** — not in commits, PRs, code comments, or any artifact. Plain `git commit` (repo-local `fabzter` identity is already pinned).
6. **Do not push** until Task 5 (which has the gh-identity guard).
7. **This is a spike, not the port.** It must NOT touch `src/Grafico/`, `src/Aplicacion/`, or any game logic. It only adds files under `spikes/d2_render/`, edits `flake.nix`, `spikes/CMakeLists.txt`, and commits converted assets under `conf/skin_default/`.
8. **IrrlichtMt API facts (already verified against the installed headers — do not "fix" these):**
   - Screenshot API is **`driver->createScreenShot()`** (returns `video::IImage*`), NOT `createScreenShotFromBackBuffer()` (that was removed in the fork). Then `driver->writeImageToFile(img, "path.png")` (format inferred from extension), then `img->drop()`.
   - `createDevice(video::EDT_OPENGL, core::dimension2d<u32>(W,H), 16, false, false, false, nullptr)` works (engine spike proves it).
   - **Mesh loading (VERIFIED in Task 1 — the fork has NO `getMesh(const char*)` / `getMesh(const io::path&)` overload; the ONLY overload is `getMesh(io::IReadFile*)`).** Load via the filesystem:
     ```cpp
     // helper used throughout the spike:
     scene::IAnimatedMesh* loadMesh(IrrlichtDevice* dev, const char* path) {
         io::IReadFile* rf = dev->getFileSystem()->createAndOpenFile(path);
         if (!rf) return nullptr;
         scene::IAnimatedMesh* mesh = dev->getSceneManager()->getMesh(rf);
         rf->drop();
         return mesh;   // null on failure
     }
     ```
     Then `smgr->addAnimatedMeshSceneNode(mesh)` → `scene::IAnimatedMeshSceneNode*`. `smgr->addCameraSceneNode()`. `node->setPosition(core::vector3df)`, `node->setScale(...)`, `node->setMaterialFlag(video::EMF_LIGHTING, false)`.
   - **Verified mesh extents (Task 1):** Tablero ≈657×63×650, Celda ≈50×10×50, Mono/MonoBlanco ≈52×110×51, Barrera ≈10×115×110. The board is ~10× the size of a cell — so a cell-to-cell `spacing` near the cell width (~55) tiles the 9×9 grid across the board, and the camera must be framed for a ~600-unit-wide scene. These are starting values; tune via the vision checks.
   - The fork has **no 3DS loader** — only `.x`, `.b3d`, `.obj`. That is why Task 1 converts assets.
9. **Reglas construction (ground truth from the working `src/Consola/main.cpp` — use exactly this shape):**
   ```cpp
   Reglas::Tablero t;                                  // default ctor builds the 9x9 grid + Grafo
   std::vector<Reglas::Jugador*> jugadores;
   jugadores.push_back(new Reglas::Jugador(0, nullptr)); // Agente* may be null — never deref'd by setJugadores/moverJugador
   jugadores.push_back(new Reglas::Jugador(1, nullptr));
   t.setJugadores(jugadores);                           // places p0 at cell (4,0), p1 at cell (4,8); marks them colocado
   ```
   - `Tablero::size_x == 9`, `size_y == 9`. `t.getCelda(x,y)` returns `datos[y][x]` (row-major, y is row).
   - `t.moverJugador(0, 4, 1)` moves player 0 to cell (4,1). It validates `estaColocado()` (true after `setJugadores`) but does NOT check move legality (that's the Juez, which the spike does not use) — so the move won't throw.
   - `t.getCelda(idJugador)` returns the cell the player currently occupies; `Celda::getPosicion()` returns `std::vector<int>` `{x,y}`.

If a step does not behave as documented, report **BLOCKED with the exact command + output**. Do NOT mock, stub, or change game logic to force a green result — surfacing a real obstacle is the spike's job.

- [ ] **Step 3.2: Implement `scene_builder.cpp`**

```cpp
#include "scene_builder.hpp"
#include <Reglas/Tablero.hpp>
#include <Reglas/Celda.hpp>
#include <vector>

using namespace irr;

namespace d2spike {

core::vector3df cellToWorld(int x, int y) {
    // Centred grid so cell (4,4) sits at the world origin — pairs with the
    // board being recentred on the origin in buildScene (Task 2 finding:
    // the board's model origin is a far corner). board ~657 wide / 9 ≈ 73.
    const float spacing = 70.0f;   // tune visually
    return core::vector3df((x - 4) * spacing, 0.0f, (y - 4) * spacing);
}

// Takes the IrrlichtDevice because the fork exposes getFileSystem() on
// IrrlichtDevice, NOT on ISceneManager (verified D2.0 Task 1).
static scene::IAnimatedMeshSceneNode* addMesh(
        IrrlichtDevice* dev, const char* path, const core::vector3df& pos) {
    scene::ISceneManager* smgr = dev->getSceneManager();
    io::IReadFile* rf = dev->getFileSystem()->createAndOpenFile(path);
    scene::IAnimatedMesh* mesh = rf ? smgr->getMesh(rf) : nullptr;
    if (rf) rf->drop();
    if (!mesh) return nullptr;
    scene::IAnimatedMeshSceneNode* node = smgr->addAnimatedMeshSceneNode(mesh);
    if (node) {
        // IrrlichtMt removed setMaterialFlag/EMF_* (Task 2 finding) — set the
        // SMaterial::Lighting bool directly via the fork's forEachMaterial.
        node->forEachMaterial([](video::SMaterial& mat) { mat.Lighting = false; });
        node->setPosition(pos);
    }
    return node;
}

void placePieceAtCell(scene::IAnimatedMeshSceneNode* node, int x, int y) {
    if (node) node->setPosition(cellToWorld(x, y) + core::vector3df(0, 10, 0));
}

SceneRefs buildScene(IrrlichtDevice* dev, const Reglas::Tablero& t) {
    SceneRefs refs;

    // Board — recentre on the world origin (its model origin is a far corner,
    // Task 2 finding) so the centred cell grid lands on it.
    scene::IAnimatedMeshSceneNode* boardNode =
        addMesh(dev, "conf/skin_default/Tablero.obj", core::vector3df(0, 0, 0));
    if (boardNode) {
        core::aabbox3df bb = boardNode->getBoundingBox();   // local-space bbox
        boardNode->setPosition(-bb.getCenter());            // board centre -> world origin
    }

    // A Celda node at every grid position (9x9).
    for (int y = 0; y < Reglas::Tablero::size_y; ++y) {
        for (int x = 0; x < Reglas::Tablero::size_x; ++x) {
            addMesh(dev, "conf/skin_default/Celda.obj", cellToWorld(x, y));
        }
    }

    // Players at their current cells (derived from real Tablero state).
    std::vector<int> p0 = t.getCelda(0).getPosicion(); // {x,y}
    std::vector<int> p1 = t.getCelda(1).getPosicion();
    refs.piece0 = addMesh(dev, "conf/skin_default/Mono.obj",
                          cellToWorld(p0.at(0), p0.at(1)) + core::vector3df(0, 10, 0));
    refs.piece1 = addMesh(dev, "conf/skin_default/MonoBlanco.obj",
                          cellToWorld(p1.at(0), p1.at(1)) + core::vector3df(0, 10, 0));

    // One barrier between cells (4,3) and (4,4) — purely to prove the barrier
    // mesh renders; placed at the midpoint, lifted slightly.
    core::vector3df a = cellToWorld(4, 3);
```
